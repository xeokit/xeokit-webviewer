## 1. Introduction

---

The xeokit SDK is an open-source toolkit for real-time 3D visualization in web-based architecture, engineering, 
and construction (AEC) applications. It enables interactive exploration of large 3D models with double-precision 
coordinates to prevent floating-point errors.

The SDK's doc:Viewer provides a flexible rendering engine supporting multiple views, object interaction, and 
lighting configurations. The doc:Scene module organizes 3D geometry, materials, and model structures, while the 
doc:Data module integrates semantic metadata for querying and analysis.

Key features include slicing, ambient occlusion, edge enhancement, BIM and BCF interoperability, configurable 
render modes, and object picking. Performance optimizations, such as adaptive resolution scaling and custom 
rendering pipelines, support smooth interaction with complex models.

This article provides an introduction to the SDK’s architecture, rendering techniques, and applications in 
web-based 3D visualization.

## 3. xeokit SDK Features

- **Graphics Programming Toolkit for AEC** - A complete 3D programming toolkit for building, loading, converting, saving, viewing and interacting with 3D building models.
- **View Multiple Models** - View multiple models alongside each other. View federated models.
- **View Multiple AEC Formats** - View IFC, glTF, LAS/LAZ, CityJSON, XKT, XGF, .BIM and more.
- **Multiple Canvases** - View models in multiple canvases, each with their own object visibilities, effects etc.
- **Double-Precision Coordinates** - View models in the Global coordinate system. View distantly-placed models without 
needing to center them to avoid jitter. Automatically aligns globally-positioned building models 
and LiDAR scans.
- **Fast Loading** - Option to pre-convert large AEC models to xeokit's native XGF format, for optimized loading.
- **Efficient Rendering** - A 3D engine build specially for viewing complex AEC models.
- **Emphasis Effects** - Slice, X-ray, highlight, select etc.
- **Natively TypeScript** - Robust components with clearly defined contracts.

## 4. xeokit SDK Modules

---

The **xeokit SDK** offers a comprehensive set of modules for managing, visualizing, and
converting 3D AECO models in both web and Node.js environments. Designed with **Separation of Concerns**,
**Single Responsibility**, and the **Open-Closed Principle**, the SDK provides a flexible toolkit of
components that applications can leverage to support a wide range of use cases.

The SDK is organized into the following module categories:

- **Scene** – A container for 3D model objects, compatible with both Node.js and browsers.
- **Data** – A searchable entity-relationship data model.
- **Viewer** – Attaches to a Scene for interactive viewing in the browser.
- **Importers** – Load supported AECO model formats into Scene and Data.
- **Exporters** – Save Scene and Data in supported AECO formats.
- **Converters** – CLI tools for converting between AECO model formats.
- **Utilities** – Low-level functions and classes for 3D graphics, math, and more.

<br>

### 4.2 xeokit Viewer

The SDK features a high-performance 3D web viewer, implemented by the doc:Viewer class, for viewing scene models in the
browser. With
a pluggable renderer strategy, it can be adapted to any browser graphics API, including WebGL and WebGPU. The viewer
enables
simultaneous visualization of multiple models, and allows the creation of multiple separate canvases with customizable
elements
such as lights, section planes, and annotations, providing flexible multi-view model display.

### 5.1. Visualize a glTF Model

[![](holterTower.png)](/packages/demos/examples/viewer/#SceneModel_build_table/index.html)

* *[Run this example](/packages/demos/examples/viewer/#SceneModel_build_table/index.html)*

First install the npm module:

````bash
npm install @xeokit/sdk
````

Then create an HTML page in `index.html` that contains a canvas element:

````html
<!DOCTYPE html>
<html>
<head>
    <title>xeokit glTF Model Viewer</title>
</head>
<body>
<canvas id="myView1"></canvas>
</body>
<script type="module" src="./index.js"></script>
````

Then create JavaScript in `index.js` to view a glTF model in the canvas.

1. Import the modules we need.
2. Create a doc:Viewer with a doc:WebGLRenderer and a doc:Scene.
3. In the Viewer, create a doc:View that draws to the canvas.
4. Position the View's doc:Camera to look at the center of the 3D coordinate system (default).
5. Create a doc:SceneModel in the Scene.
6. Use doc:loadGLTF to load a glTF building model into the SceneModel.
7. Build the SceneModel. This causes the SceneMode's objects to appear in the View's canvas.
8. Update the visibility and highlight status of an object in the model.
9. Orbit the Camera by one degree on each Viewer "tick" event. This will continually orbit the model.

````javascript
// 1.
import {Scene} from "@xeokit/sdk/scene";
import {Viewer} from "@xeokit/sdk/viewer";
import {WebGLRenderer} from "@xeokit/sdk/webglrenderer";
import {loadGLTF} from "@xeokit/sdk/gltf";

// 2.
const scene = new Scene();
const renderer = new WebGLRenderer({});
const viewer = new Viewer({scene, renderer});

// 3.
const view = myViewer.createView({id: "myView", canvasId: "myView1"});

// 4.
view.camera.eye = [0, 0, 10]; // Looking down the -Z axis
view.camera.look = [0, 0, 0];
view.camera.up = [0, 1, 0];

// 5.
const sceneModel = scene.createModel();

// 6.
fetch("myModel.glb").then(response => {
    response.arrayBuffer().then(fileData => {
        loadGLTF({fileData, sceneModel}).then(() => {

            // 7.
            sceneModel.build().then(() => {

                // 8.
                view.objects["2hExBg8jj4NRG6zzE$aSi6"].visible = true;
                view.objects["2hExBg8jj4NRG6zzE$aSi6"].highlighted = false;  // etc.

                // 9.
                viewer.onTick.subscribe(() => {
                    view.camera.orbitYaw(1.0);
                });
            });
        });
    });
});
````
